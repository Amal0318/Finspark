# SentinelX AI - Feature Engineering

This document explains the mathematical formulas and cybersecurity logic of the engineered features generated by the SentinelX pipeline.

---

## 1. Temporal & Behavioral Indicators

*   **`working_hour`:** Binary flag set to `1` if the activity timestamp hour is between 9:00 AM and 5:00 PM local time. Detects abnormal late-night accesses.
*   **`weekend_login`:** Binary flag set to `1` if logons occur on Saturday or Sunday.
*   **`abnormal_login_time`:** Flag indicating activities occurring between 10:00 PM and 6:00 AM (high-risk window).

---

## 2. Authentication & Device Profiles

*   **`is_new_device`:** Flags a device fingerprint seen for the very first time for that specific user.
*   **`vpn_detected`:** Flags geocoordinate jumps paired with network round-trip latency spikes (> 350ms).
*   **`failed_login_count`:** Rolling 10-minute counter tracking bad authentication attempts.
*   **`multiple_failed_login`:** Binary flag set to `1` if `failed_login_count` is $\ge 3$. Detects active brute-force attempts.

---

## 3. Financial Outlier Transactions

*   **`transaction_velocity`:** Count of transactions initiated by the client within a rolling 1-hour window.
*   **`average_transaction`:** Cumulative historical mean of transaction sizes for this account.
*   **`transaction_deviation`:** Absolute difference: $\text{Deviation} = |\text{Current Amount} - \text{Average Amount}|$.
*   **`high_value_transaction`:** Set to `1` if the transaction size is $> \$5,000.00$.

---

## 4. Insider Activity Scores (CERT-driven)

*   **`usb_activity_score`:** Min-max normalized count (0-10) of USB storage connects.
*   **`email_activity_score`:** Min-max normalized rating of outbound email attachments and transfer sizes.
*   **`file_access_score`:** Min-max normalized count of file copying and modifications.
